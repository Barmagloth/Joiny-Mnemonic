Создать полноценный агент-агностичный инструмент памяти для длительных LLM-сессий.

Цель продукта:
Обеспечить непрерывную работу агента между сессиями при ограниченном контекстном окне, без потери исходных данных, инструкций, решений и состояния проекта.

Взять лучшие решения:
- OpenHands: append-only History и отдельный сжатый View.
- CMV: immutable snapshots, branching и lineage DAG.
- claude-mem: progressive disclosure — index → timeline → detail → source.
- Letta: постоянно доступные memory blocks и archival memory.
- agentmemory: типизированные facts/decisions/tasks/preferences и provenance.
- codegraph: получение точного актуального содержимого из источника.
- AdaHiMem: multi-resolution memory, budget governor, promotion/demotion и физические tiers.

Обязательные функции:

1. Каноническая память
- Точный append-only журнал сообщений, tool calls, outputs и артефактов.
- Запрет замены или удаления единственной исходной копии.
- Версионирование и provenance всех производных записей.

2. Активная память
- Защищённые блоки: инструкции, цель, ограничения, решения, открытые задачи.
- Последние сообщения дословно.
- Компактный индекс старой истории.
- Автоматическая сборка prompt под заданный token budget.

3. Многоуровневые представления
- Index.
- Структурированное состояние.
- Summary.
- Точный источник.
- Возможность добавлять embeddings, knowledge graph и KV-представления как плагины.

4. Retrieval и promotion
- Поиск по времени, тексту, семантике, файлам и типам памяти.
- Возврат точного источника, когда summary недостаточно.
- Контекстная оценка полезности с учётом запроса, свежести, риска и стоимости.
- Никакой постоянной универсальной importance-оценки.

5. Snapshots
- Атомарные incremental snapshots.
- Восстановление через snapshot + replay хвоста журнала.
- Parent/branch lineage.
- Компактный resume packet для новой сессии.
- Привязка к Git HEAD и hashes файлов; обнаружение устаревшего состояния.

6. Agent-agnostic интеграция
- Независимое ядро с MCP, CLI и API.
- Адаптеры минимум для Claude Code, Codex и OpenCode/OpenHands.
- Capability model: функции gracefully отключаются, если агент не предоставляет нужные hooks или KV-доступ.

7. Физическая память
- Для API-агентов: текст, токены, summaries, retrieval и recompute.
- Для self-hosted inference: GPU/CPU KV, квантизация, offload и повторный prefill.
- Выбор хранения или recompute по памяти, latency и частоте использования.

8. Безопасность
- Retrieved memory всегда считается данными, а не инструкциями.
- Активные инструкции защищены от compaction и prompt injection.
- Секреты фильтруются до сохранения.

Не включать:
- hard eviction канонических данных;
- генеративное восстановление точных фактов;
- обязательный knowledge graph;
- attention score как источник истины;
- сложный positional/semantic/attention super-DAG;
- SVD или конкретный KV-компрессор в ядре.

Критерии готовности:
- Ни одно подтверждённое событие не теряется после аварийного завершения.
- Новая сессия получает достаточное состояние в ≤1500 стартовых токенов.
- Все активные инструкции сохраняются.
- Любое утверждение памяти трассируется до источника.
- Устаревшие snapshots обнаруживаются автоматически.
- Resume-задачи достигают не менее 95% качества full-history baseline.
- Одинаковое ядро работает минимум с тремя агентами.
- Все политики сравниваются на task-level качестве, token cost, latency и объёме хранения.
